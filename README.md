# DOOMSCROLL

A retro pseudo-3D shooter for **portrait-mode phones**, in the spirit of the
original Doom (1993). Runs in the browser — no install, no build step, no
frameworks, no asset files: every texture, sprite and sound is generated by
code at load time.

## Play

Serve the folder over HTTP and open it on a phone (or a narrow browser
window):

```bash
python3 -m http.server 8000
# open http://localhost:8000
```

## Controls

Designed to be used **without looking at your thumbs** — the controls appear
wherever you touch:

- **Right thumb** (anywhere on the right): a joystick appears where you land.
  Up/down walks, left/right turns.
- **Left thumb** (anywhere on the left): a fire button appears where you land.
  Hold to keep firing.
- **Bottom status bar**: swipe sideways and hold to strafe.
- **Doors open when you shoot them.** So does the exit switch.
- Tap the **ARMS 1/2/3** grid on the status bar to switch weapons.
- Desktop: WASD/arrows move, Q/E strafe, Space fires, 1-3 switch weapons.

## The game

Zone 1 — *Hangar 13*: a start hangar, a zigzag nukage room with an armour
platform, a computer room, an open-sky courtyard with a toxic moat (something
useful is on the island), and an exit switch behind the last door.

- **3 enemy types**: rifle grunts, shotgun sergeants, and fireball-throwing
  brutes.
- **3 weapons**: you start with the pistol; the shotgun and chaingun are
  hidden in the level.
- Exploding barrels, dropped ammo clips, medkits, armour, a reactive
  status-bar face that takes damage as you do — the works.

## Code map

| file | what it does |
|---|---|
| `js/main.js` | boot, resize, game loop, screen states |
| `js/engine.js` | raycaster: walls, floors/ceilings/sky, sprite billboards |
| `js/game.js` | simulation: player, enemies, weapons, doors, pickups |
| `js/level.js` | the map (ASCII grid — every row exactly 32 chars) |
| `js/sprites.js` | all pixel art, drawn in code |
| `js/textures.js` | procedural wall/floor/sky textures |
| `js/hud.js` | status bar, weapon overlay, touch UI, title/win/death screens |
| `js/input.js` | floating joystick / fire button / strafe swipe / keyboard |
| `js/audio.js` | synthesised sound effects (WebAudio, no files) |
| `js/px.js` | tiny pixel-buffer toolkit shared by the art generators |

Dev console helpers: `DS.start()`, `DS.warp(x, y, angleDeg)`, `DS.game`.
